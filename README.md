# auto-sim-exp

> **Antigravity Skill** · 全自动 UAV 仿真实验执行器

通过自然语言指令，让 AI 助手自主完成从 SSH 连接、Docker 启动、ROS/PX4 环境搭建，到仿真飞行、截图采集、会话清理的完整实验流程。无需人工干预，一句话触发，全程自动执行。

---

## ✨ 功能概览

| 能力 | 描述 |
|------|------|
| 🔌 远程连接管理 | 自动建立 SSH 会话，全程复用同一连接 |
| 🐳 Docker 生命周期 | 自动启动容器、进入容器、实验结束后退出 |
| 🚀 仿真环境搭建 | 自动 source ROS / PX4 工作空间，拉起 Gazebo + RViz |
| ✈️ 飞行过程监控 | 等待无人机完成起飞-降落序列，期间不干扰终端 |
| 📸 自动截图 | 仿真进行 20 秒时远程截图并传回本地桌面 |
| 🧹 会话清理 | 发送多次中断信号，强制终止仿真，清理退出所有层级 |

---

## 🚀 快速开始

在 Antigravity 对话框中，对 AI 说任意一种触发语句：

```
帮我跑一次仿真实验
Run a simulation experiment
/run_simulation_experiment
```

AI 将**全程自主执行**，无需你输入任何命令。关键里程碑（仿真已启动、飞行中、已退出）会实时汇报进度。

## ⚙️ 当前实验流程

```
SSH 连接
  └── 进入部署目录
       └── 启动 Docker 容器
            └── 进入容器
                 └── Source ROS 工作空间
                      └── Source PX4 环境
                           └── roslaunch 仿真场景
                                ├── [20s] 截图并传回本地桌面
                                └── [30s 后] Ctrl+C 停止仿真
                                     └── 退出 Docker → 退出 SSH
```

## 🗺️ TODO · 未来规划

以下是计划中的能力增强方向，欢迎贡献：

### 🤖 智能化 / 全自动化

- [ ] **自动识别实验入口**：扫描项目目录，自动找到可执行的 `roslaunch` / `launch.py` / `run.sh` 等入口文件，无需在 SKILL.md 中硬编码路径
- [ ] **自动解析实验参数**：读取 `launch` 文件或配置文件（YAML/JSON），自动推断可配置参数（场景、检查点路径、飞行时间等），支持用户用自然语言覆盖
- [ ] **多场景批量实验**：支持一次性指定多个场景或参数组合，顺序/并行执行，生成对比报告
- [ ] **实验进度感知**：通过解析 ROS topic 或日志，判断无人机是否真正起飞/降落，而非固定等待时长

### 📊 实验结果分析

- [ ] **自动收集 ROS bag**：仿真结束后自动录制或下载 `.bag` 文件到本地
- [ ] **飞行轨迹可视化**：解析 bag 文件，生成轨迹图、速度时序图等可视化报告
- [ ] **成功率统计**：多次实验后自动汇总成功/失败/碰撞率，输出 Markdown 报告
- [ ] **与超参数调优联动**：接入 `auto-hparam-tuning` skill，将仿真指标（飞行成功率、平均速度等）作为优化目标

### 🤝 实机操控实验

- [ ] **实机迁移模式**：在仿真验证通过后，一键切换为实机部署流程（修改 launch 参数、连接飞控、安全检查）
- [ ] **实机飞行监控**：通过 MAVLink / ROS topic 实时监控飞机状态，异常时自动触发返航
- [ ] **仿真 vs 实机对比**：自动对比仿真与实机的飞行数据差异，输出 Sim2Real Gap 报告

### 🔧 工程改进

- [ ] **连接配置外置**：将 SSH 主机名、部署路径、Docker 脚本等配置项提取到独立配置文件，避免修改 SKILL.md
- [ ] **多服务器支持**：支持选择不同的远程主机（如 `zgc5090`、`dgx-node` 等）执行实验
- [ ] **实验日志归档**：自动将每次实验的终端输出、截图、时间戳归档到本地 `experiments/` 目录

## 📋 依赖说明

| 依赖项 | 说明 |
|--------|------|
| SSH 配置 | 本地 `~/.ssh/config` 中需配置好目标主机别名 |
| `scrot` | 远程主机上需安装截图工具（用于自动截图步骤） |
| `scp` | 本地环境需支持 `scp` 命令（截图文件传输） |
| ROS / PX4 | 远程 Docker 容器内已预装 |
